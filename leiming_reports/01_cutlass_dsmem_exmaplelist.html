CUTLASS Examples Using Distributed Shared Memory (DSMEM)

This report summarizes a search of cutlass/examples for explicit use of distributed shared memory (DSMEM) — i.e., code that directly reads or writes another CTA's shared memory within a thread-block cluster (via mapa / st.async.shared::cluster PTX), as opposed to code that merely uses cluster shapes for TMA multicast loads or 2-SM cooperative MMA.

Examples Using DSMEM

Example File Details
93_blackwell_low_latency_gqa common.cuh (line 60) store_shared_remote_f32() — uses st.async.shared::cluster.mbarrier::complete_tx.bytes PTX to write directly into a remote CTA's shared memory.
common.cuh (line 68) get_dsmem_tensor() — uses mapa (via set_block_rank) to convert a local smem tensor pointer into a remote CTA's dsmem address.
tgv_gqa.cuh (lines 1105, 1106, 1122, 1498, 1499, 1525) Uses the DSMEM helpers above to build cross-CTA "mailboxes" for flash-decoding cluster reduction (exchanging softmax max/sum and partial output accumulators between CTAs in a cluster).
tgv_gqa_paged.cuh, readme.md Reference the same DSMEM helpers for the paged-KV-cache variant of the kernel.
Conclusion: 93_blackwell_low_latency_gqa is the only example in cutlass/examples found to use true DSMEM (explicit cross-CTA shared-memory addressing).

Related but Not True DSMEM

Many Hopper/Blackwell examples use thread-block clusters (ClusterShape, cluster.sync()) but only for TMA multicast loads or 2-SM cooperative MMA — not for direct cross-CTA shared-memory addressing:

  • 48_hopper_warp_specialized_gemm/48_hopper_warp_specialized_gemm.cu
  • 70_blackwell_gemm/70_blackwell_fp16_gemm.cu
  • 73_blackwell_gemm_preferred_cluster/blackwell_gemm_preferred_cluster.cu
  • cute/tutorial/blackwell/03_mma_tma_multicast_sm100.cu
  • cute/tutorial/blackwell/04_mma_tma_2sm_sm100.cu

These examples rely on cluster-level synchronization primitives, but do not perform direct DSMEM store/load addressing the way 93_blackwell_low_latency_gqa does. It is possible that CUTLASS's internal pipeline/barrier classes (used under the hood by these cluster-based GEMM/FMHA examples) touch DSMEM, but this was not confirmed at the explicit-usage level searched here.
