What These Examples Do

Both examples implement a GEMM: D (F32) = β·C (F32) + α·A (F16) × B (F16) using low-level CuTe SM100 primitives, progressively building on each other:

  • Tutorial 03 introduces TMA multicast: a single TMA load broadcasts a tile of A (or B) to all CTAs in the cluster that share the same M (or N) coordinate, eliminating redundant GMEM reads. The cluster is 4×4 CTAs; each CTA runs a 1-SM tcgen05.mma (128×256×16 MMA tile).
  • Tutorial 04 adds 2-SM cooperative MMA: pairs of CTAs collaborate on a single 256×256×16 tcgen05.mma instruction using SM100_TMA_2SM_LOAD_MULTICAST and SM100_MMA_2SM_F16BF16_SS. Only the leader CTA issues the MMA; both CTAs contribute SMEM data.

Why They Cannot Be Run Naively on SM 11.0

#ProblemRoot cause
1 Runtime check (props.major != 10) || (props.major == 10 && props.minor > 1) rejects SM 11.0. Guard written for Blackwell SM 100 only; Thor has major=11.
2 cudaErrorInvalidClusterSize at launch: the hardcoded 4×4×1 cluster (16 CTAs) exceeds SM 11.0's maximum. SM 11.0 supports clusters up to 8 CTAs (the portable limit). SM 100 (Blackwell) supports up to 16 CTAs with an explicit opt-in. The examples use 16 without the opt-in flag, which fails on SM 11.0.
3 CMake 3.28 cannot generate correct arch flags for SM 11.0. Same zero-padding issue as all previous examples; bypassed by invoking nvcc directly.
No kernel-level changes needed. The TCGEN05 MMA instructions, TMA multicast, and TMEM allocator used by both examples are all supported on SM 11.0a. The only source changes are the runtime arch check and the cluster shape.

Maximum Cluster Size on SM 11.0

Empirically verified by attempting cluster launches of increasing size:

Cluster size (total CTAs)SM 11.0 result
1 – 8✓ Launch succeeds
9 – 16✗ cudaErrorInvalidClusterSize

The original 4×4×1 = 16 cluster is replaced with 2×2×1 = 4, which:

  • Is within the SM 11.0 portable limit of 8.
  • Keeps the cluster M dimension a multiple of 2, satisfying the 2-SM MMA constraint in tutorial 04.
  • Preserves the multicast semantics: A tiles are still multicast along the N-cluster dimension (now 2 instead of 4), and B tiles along the M-cluster dimension (now 2).

Step-by-Step Build Instructions

1 Patch both source files

# Back up originals
TDIR=cutlass/examples/cute/tutorial/blackwell
cp $TDIR/03_mma_tma_multicast_sm100.cu $TDIR/03_mma_tma_multicast_sm100.cu.bak
cp $TDIR/04_mma_tma_2sm_sm100.cu       $TDIR/04_mma_tma_2sm_sm100.cu.bak

Apply the following diff to both files (line numbers differ slightly):

--- 03_mma_tma_multicast_sm100.cu.bak  (same pattern in 04_mma_tma_2sm_sm100.cu.bak)
+++ 03_mma_tma_multicast_sm100.cu

@@ cluster shape (line 524 in 03, line 526 in 04) @@
-  auto cluster_shape = make_shape(Int<4>{}, Int<4>{}, Int<1>{});
+  auto cluster_shape = make_shape(Int<2>{}, Int<2>{}, Int<1>{});

@@ runtime arch check (line 615 in 03, line 622 in 04) @@
-  if ((props.major != 10) || (props.major == 10 && props.minor > 1)) {
-    std::cerr << "This example requires NVIDIA's Blackwell Architecture GPU with compute capability 100a." << std::endl;
-    std::cerr << "  Found " << props.major << "." << props.minor << std::endl;
+  if (props.major < 10) {
+    std::cerr << "This example requires a GPU with compute capability >= 100 (SM 100 or SM 110)." << std::endl;

2 Build

CUTLASS_DIR=/home/leimyu/01_research/cutlass_dsmem/cutlass
BUILD_DIR=/home/leimyu/01_research/cutlass_dsmem/build_sm110
TDIR=$CUTLASS_DIR/examples/cute/tutorial/blackwell
mkdir -p $BUILD_DIR/examples/cute/tutorial/blackwell

NVCC_FLAGS="-std=c++17 -O3 \
  --generate-code=arch=compute_110a,code=[sm_110a,compute_110a] \
  --expt-relaxed-constexpr \
  -lineinfo \
  -ftemplate-backtrace-limit=0 \
  -DCUTLASS_ENABLE_TENSOR_CORE_MMA=1 \
  -DCUTLASS_ENABLE_GDC_FOR_SM100=1"

INCLUDES="-I$CUTLASS_DIR/include \
  -I$CUTLASS_DIR/tools/util/include \
  -I$CUTLASS_DIR/examples/common \
  -I$TDIR"

# Tutorial 03
nvcc $NVCC_FLAGS $INCLUDES \
  $TDIR/03_mma_tma_multicast_sm100.cu \
  -o $BUILD_DIR/examples/cute/tutorial/blackwell/03_mma_tma_multicast_sm100 \
  -lcuda

# Tutorial 04
nvcc $NVCC_FLAGS $INCLUDES \
  $TDIR/04_mma_tma_2sm_sm100.cu \
  -o $BUILD_DIR/examples/cute/tutorial/blackwell/04_mma_tma_2sm_sm100 \
  -lcuda

3 Run

BIN=$BUILD_DIR/examples/cute/tutorial/blackwell

# Tutorial 03 — default and larger problem
$BIN/03_mma_tma_multicast_sm100
$BIN/03_mma_tma_multicast_sm100 1024 2048 512

# Tutorial 04 — default and larger problem
$BIN/04_mma_tma_2sm_sm100
$BIN/04_mma_tma_2sm_sm100 1024 2048 512
