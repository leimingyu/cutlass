Build Report: 70_blackwell_fp16_gemm on NVIDIA Thor (SM 11.0)

This report documents building and running cutlass/examples/70_blackwell_gemm/70_blackwell_fp16_gemm.cu on an NVIDIA Thor GPU (SM 11.0) using CUDA 13.1. It also consolidates the build of 48_hopper_warp_specialized_gemm into a new, clean build_sm110/ directory so that both examples share identical compiler flags and can be compared side-by-side.

Impact on the Previous Build of Example 48

The previous build report (build_48_hopper_warp_specialized_gemm.html) placed the example 48 binary in build/examples/48_hopper_warp_specialized_gemm/ using an ad-hoc nvcc invocation. Building example 70 alongside it motivates a unified, clean build tree (build_sm110/) so that:

  • Both examples are compiled with identical flags and the same sm_110a target.
  • The build is reproducible from scratch without relying on the earlier CMake-generated build/ tree.
  • Future examples can be added to build_sm110/ with the same one-liner pattern.
Example 48 source is unchanged from the previous build report. The only difference is the output path: build_sm110/ instead of build/. The patched source (48_hopper_warp_specialized_gemm.cu) already targets Sm100 / CUTLASS_ARCH_MMA_SM100_SUPPORTED and is correct for SM 11.0.

Why Example 70 Cannot Be Run Naively on SM 11.0

#ProblemRoot cause
1 Runtime check props.major != 10 || props.minor != 0 rejects SM 11.0. The guard was written for Blackwell SM 100 only; Thor SM 11.0 has major=11.
2 CMake 3.28 cannot generate correct arch flags for SM 11.0 (same issue as example 48). CMake 3.28 zero-pads the arch number, producing the invalid compute_1100.
Note: The CUDA version check (__CUDACC_VER_MAJOR__ < 12 || (__CUDACC_VER_MAJOR__ == 12 && __CUDACC_VER_MINOR__ < 8)) already passes with CUDA 13.1 and requires no change. The kernel itself (CUTLASS_ARCH_MMA_SM100_SUPPORTED, Sm100 arch tag, Shape<int,int,int,int> problem shape) is already correct for SM 11.0 when compiled with sm_110a — no template changes are needed.

Step-by-Step Build Instructions

1 Verify the environment

nvidia-smi --query-gpu=name,compute_cap --format=csv,noheader
# Expected: NVIDIA Thor, 11.0

nvcc --version
# Expected: release 13.1 or newer

2 Patch 70_blackwell_fp16_gemm.cu

Back up the original and apply the single runtime-check fix. No template or kernel-configuration changes are required. 

# Back up
cp cutlass/examples/70_blackwell_gemm/70_blackwell_fp16_gemm.cu \
   cutlass/examples/70_blackwell_gemm/70_blackwell_fp16_gemm.cu.bak

Apply the following diff:

--- 70_blackwell_fp16_gemm.cu.bak
+++ 70_blackwell_fp16_gemm.cu

@@ line 458 — runtime arch check @@
-  if (props.major != 10 || props.minor != 0) {
-    std::cerr << "This example requires a GPU with compute capability 100a)." << std::endl;
+  if (props.major < 10) {
+    std::cerr << "This example requires a GPU with compute capability >= 100 (SM 100 or SM 110)." << std::endl;

This is the only change needed. The kernel configuration (ArchTag = Sm100, MmaTileShape_MNK = Shape<_256,_128,_64>, ClusterShape_MNK = Shape<_2,_2,_1>, 4-D problem shape) is already correct for SM 11.0.

3 Create the unified build_sm110/ directory

mkdir -p build_sm110/examples/48_hopper_warp_specialized_gemm
mkdir -p build_sm110/examples/70_blackwell_gemm

4 Define shared compiler variables

Both examples use identical flags. Define them once to avoid repetition and guarantee consistency. 

CUTLASS_DIR=/home/leimyu/01_research/cutlass_dsmem/cutlass
BUILD_DIR=/home/leimyu/01_research/cutlass_dsmem/build_sm110

NVCC_FLAGS="-std=c++17 -O3 \
  --generate-code=arch=compute_110a,code=[sm_110a,compute_110a] \
  --expt-relaxed-constexpr \
  -lineinfo \
  -ftemplate-backtrace-limit=0 \
  -DCUTLASS_ENABLE_TENSOR_CORE_MMA=1 \
  -DCUTLASS_ENABLE_GDC_FOR_SM100=1"

INCLUDES="-I$CUTLASS_DIR/include \
  -I$CUTLASS_DIR/tools/util/include \
  -I$CUTLASS_DIR/examples/common"

5 Build example 48 (TF32 GEMM, re-built in clean tree)

nvcc $NVCC_FLAGS $INCLUDES \
  -I$CUTLASS_DIR/examples/48_hopper_warp_specialized_gemm \
  $CUTLASS_DIR/examples/48_hopper_warp_specialized_gemm/48_hopper_warp_specialized_gemm.cu \
  -o $BUILD_DIR/examples/48_hopper_warp_specialized_gemm/48_hopper_warp_specialized_gemm \
  -lcuda

6 Build example 70 (FP16 GEMM)

nvcc $NVCC_FLAGS $INCLUDES \
  -I$CUTLASS_DIR/examples/70_blackwell_gemm \
  $CUTLASS_DIR/examples/70_blackwell_gemm/70_blackwell_fp16_gemm.cu \
  -o $BUILD_DIR/examples/70_blackwell_gemm/70_blackwell_fp16_gemm \
  -lcuda

Key flags (same for both examples):

FlagPurpose
--generate-code=arch=compute_110a,code=[sm_110a,compute_110a] Produces a native SM 11.0a cubin plus embedded PTX for forward compatibility. The 110a variant sets __CUDA_ARCH_FEAT_SM110_ALL, activating TCGEN05 MMA and TMA device-side guards inside CUTLASS headers.
--expt-relaxed-constexpr Required by CUTLASS's heavy use of constexpr in device code.
-DCUTLASS_ENABLE_TENSOR_CORE_MMA=1 Enables tensor-core (TCGEN05 / UMMA) MMA paths in CUTLASS.
-DCUTLASS_ENABLE_GDC_FOR_SM100=1 Enables Grid Dependency Control used by the SM 100/110 persistent tile scheduler.
-I$CUTLASS_DIR/examples/common Provides helper.h (GPU timer, CUDA/CUTLASS error-check macros).
-lcuda Links the CUDA driver library (needed for TMA descriptor APIs).

7 Run the examples

# Example 48 — TF32 GEMM
$BUILD_DIR/examples/48_hopper_warp_specialized_gemm/48_hopper_warp_specialized_gemm
$BUILD_DIR/examples/48_hopper_warp_specialized_gemm/48_hopper_warp_specialized_gemm \
  --m=2048 --n=2048 --k=2048 --rasterization=N --swizzle=2

# Example 70 — FP16 GEMM
$BUILD_DIR/examples/70_blackwell_gemm/70_blackwell_fp16_gemm
$BUILD_DIR/examples/70_blackwell_gemm/70_blackwell_fp16_gemm \
  --m=4096 --n=4096 --k=4096 --swizzle=2
